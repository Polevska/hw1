  • git init - to create an empty Git repository
  • git status - to display the state of repository and staging area
  • git add - to add new or changed files in your working directory to the Git staging area
  • git commit - to create a commit, which is like a snapshot of your repository; to improve your changes
  • git log - to show the commit logs
  • git checkout - to update files in the working tree to match the version in the index or the specified tree
  • git branch - to list, create or delete branches
  • git push - to upload all local branch commits to the corresponding remote branch
  • git pull - to update your current local working branch, taking this updates from the remote tracking branche
  • git merge - to join two or more development histories together, to merge changes from one branch into another
  • git clone - to create a copy of a specific repository or branch within a repository

  • .gitignore - to specify intentionally untracked files to ignore

  • git revert - to revert/change back some to some existing commits
  • git rest - to move the repository back to a previous commit , discarding any changes
  • git rebase - to reapply, reoder commits
  • git cherry-pick - to apply the changes introduced by some existing commits
  • git stash - to save the uncommitted changes locally
